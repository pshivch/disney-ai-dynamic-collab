# README_RUN — Multi‑Language Demos (Detailed)

This guide helps you run the **Java**, **Kotlin**, and **JavaScript** demos that reproduce the same visuals used in the main README.

> If you only need Python, use the main `README.md` instructions (`pip install -r requirements.txt` then `python demo.py`).

---

## 🔧 Prerequisites

### macOS
```bash
# Install Homebrew if needed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Java JDK (for Java & Kotlin)
brew install openjdk@17

# Gradle (Kotlin build tool)
brew install gradle

# Maven (Java build tool)
brew install maven

# (Optional) Python if not installed
brew install python
```

### Ubuntu / Debian Linux
```bash
sudo apt update
sudo apt install -y openjdk-17-jdk maven gradle python3 python3-pip
```

### Windows
1. Install **Java 17 JDK** from Adoptium (Temurin) or Oracle.
2. Install **Maven** and **Gradle** (use Scoop or Chocolatey):
   - Scoop: `scoop install maven gradle`
   - Chocolatey: `choco install maven gradle`
3. Ensure `java -version`, `mvn -version`, `gradle -v` work in a new terminal.

## 📂 Folder Layout

## ☕ Java (Maven + XChart)

**Files**
- `java-xchart/pom.xml`
- `java-xchart/src/main/java/com/priya/disney/ChartDemo.java`

**Run**
```bash
cd java-xchart
mvn -q -DskipTests package
mvn -q exec:java -Dexec.mainClass="com.priya.disney.ChartDemo" -Dexec.cleanupDaemonThreads=false
```

**Output**
- PNGs written to `java-xchart/out-java/`
  - `character_presence.png`
  - `tempo_ai_vs_human.png`
  - `collaboration_heatmap.png`

If `exec:java` fails, install the plugin and retry:
```bash
mvn -q dependency:get -Dartifact=org.codehaus.mojo:exec-maven-plugin:3.1.0
```

---

## 🦾 Kotlin (Gradle + XChart)

**Files**
- `kotlin-xchart/build.gradle.kts`
- `kotlin-xchart/src/main/kotlin/Charts.kt`

**Run**
```bash
cd kotlin-xchart
./gradlew run    # Windows: gradlew.bat run
```

**Output**
- PNG written to `kotlin-xchart/out-kotlin/`
  - `character_presence.png`

> You can extend `Charts.kt` to add the tempo line + heatmap, mirroring the Java example.

## 🌐 JavaScript (Chart.js)

**Files**
- `js-chartjs/index.html`

**Run (local preview)**
- Double‑click `index.html` to open in your browser, or serve with a tiny HTTP server:
```bash
cd js-chartjs
python3 -m http.server 8000
# then visit http://localhost:8000 in your browser
```

**What you’ll see**
- Character Presence (bar) and Tempo Shifts (line) rendered in‑browser.

**Optional: GitHub Pages**
- Move `js-chartjs/` to repo root and enable **Settings → Pages → Build from main** to host the charts publicly.

## 🧠 Notes for Reviewers (What’s Under the Hood)

- **Fantasia Inspiration** — music + animation “negotiate” timing; our code simulates that via multi‑agent influence + tempo curves.
- **Tempo Series** — `AI` vs `Human` generated by baseline + low‑frequency oscillations + jitter to mimic baton phrasing and rubato.
- **Heatmap** — influence matrix (0–1) across agents (Mickey, Elsa, Woody, Moana, Genie); reads like a coordination game coupling matrix.
- **Why Polyglot?** — identical logic reproduced in Python, Java, Kotlin, and JavaScript to demonstrate language breadth with consistent outputs.
